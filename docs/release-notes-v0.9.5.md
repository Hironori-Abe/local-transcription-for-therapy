# Local Transcription for Therapy (LoTT) v0.9.5

音声入力と区間聞き直しを、より速く安定して使えるようにした改善リリースです。
あわせて、AIによるセキュリティ・プライバシー監査とその指摘の改修、利用者が自分で安全性を確認するための文書追加を行いました（詳細は「セキュリティ・プライバシー」参照）。

## 主な変更点

- 初回の音声入力後もモデルを保持し、2回目以降の音声入力・区間聞き直しを高速化しました。初回はモデル読み込みの案内を表示します。
- 音声に忠実な候補、自然な表記、音として成立する別解を生成するよう改善しました。
- 区間聞き直しに前後0.25秒の余白を加え、語頭・語尾の欠けを抑えました。
- 候補を選んだときの挿入・置換動作を分かりやすく表示しました。
- マイクボタンを大きくし、押しやすくしました。
- アンインストール時に、ダウンロード済みモデルを保持できるようにしました。
- アプリアイコンを更新しました。

## Editor版のメモリ案内

- **16GB未満**: 音声入力は非推奨です。編集画面の音声入力・区間聞き直しボタンは初期状態で表示せず、設定タブで警告を確認した後に有効化できます。
- **16GB以上24GB未満**: 他のアプリがメモリを多く使用している場合の注意を表示します。
- **24GB以上**: メモリ警告は表示しません。

設定タブの「音声入力パック」は、メモリ容量にかかわらず表示されます。

## モデルの保持と解放

音声入力モデルは、次の音声入力・区間聞き直しに備えて保持されます。校正・文字起こし・話者分離を開始したとき、またはアプリを終了したときに解放されます。

Windowsでは推論プロセスをJob Objectで管理しているため、アプリが強制終了した場合も推論プロセスが終了し、使用していたRAM・VRAMはOSによって解放されます。

## セキュリティ・プライバシー

本バージョンでは、AI（Anthropic Claude Fable 5）によるコードベース全体のセキュリティ・プライバシー監査を実施しました（2026-07-16、対象: v0.9.5開発版）。観点は「会話・音声データがPC外へ出る経路の有無」「オフライン設計の抜け道」「一時ファイル・ログへのデータ残留」などです。**結果、会話・音声データがPC外へ送信される経路は発見されませんでした。** あわせて指摘された残余リスク4件（下記の「監査指摘」印）は、すべて本バージョンで改修済みです。

本バージョンに含まれるセキュリティ・プライバシー強化の一覧:

- 画面（WebView）からの通信を制限するCSPを有効化し、通信先をPC内に限定しました
- 編集画面の音声再生用ローカル配信に使い捨てトークン認証を追加し、許可した音声ファイル以外にアクセスできないようにしました
- 会話内容を含む一時ファイルを、所有者のみアクセスできるアプリ専用領域に隔離し、処理後に自動削除するようにしました（強制終了で残った場合も起動時に回収します）
- 話者分離が作る一時音声ファイルも、同じ保護領域に作成するようにしました（監査指摘）
- 音声ファイルのパス等を、他のプロセスから見えるコマンドライン引数ではなく環境変数で内部プロセスへ渡すようにしました（監査指摘・共用PC対策）
- 接続先の「localhost」指定を実接続時に127.0.0.1へ固定し、hostsファイルの改ざんで接続先が外部へ向く余地をなくしました
- 話者分離モデルの取得に使うHugging Faceトークンを、使用後にメモリ上から消去するようにしました
- 本番ビルドのCSPから、開発専用の通信許可項目を除去しました（監査指摘）
- OS・GPUドライバ等システム側の通信は本アプリの管理外である旨をREADMEに明記しました（監査指摘）

音声入力・区間聞き直し・候補生成は従来どおりPC内で完結し、音声や会話データをPC外へ送信しません。

公式インストーラーではローカルAIアプリ（LM Studio / Ollama）連携を無効化し、インストール時の選択ダイアログを廃止しました。連携が必要な場合のみ、ソースからCargo feature `local-llm-apps`付きの専用インストーラーをビルドできます。接続先は従来どおり同じPC内のloopbackに限定されます。
