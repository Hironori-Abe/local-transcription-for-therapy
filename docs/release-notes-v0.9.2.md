# Local Transcription for Therapy (LoTT) v0.9.2

臨床心理・カウンセリング会話のための、ローカル完結の日本語文字起こしデスクトップアプリです。
文字起こし・話者分離・文章校正を、会話データを PC の外へ送ることなく実行できます。

## ダウンロード

| ファイル | 対象 | 備考 |
| --- | --- | --- |
| `LoTT-v0.9.2-windows-x64-cuda-setup.exe` | NVIDIA GPU (CUDA 12.x) | 主配布・安定版 |
| `LoTT-v0.9.2-windows-x64-editor-setup.exe` | GPU 不要 | 校正中心の軽量版（文字起こし・話者分離なし） |
| `SHA256SUMS.txt` | — | 各ファイルの SHA-256 チェックサム |

ダウンロード後の検証（任意）:

```sh
sha256sum -c SHA256SUMS.txt
```

## 変更点

v0.9.1 以降の機能追加とセキュリティ強化をまとめたリリースです。既定の動作（日本語の文字起こし・Gemma 4 E4B による校正）は従来どおりで、設定を変えなければ v0.9.1 と同じように使えます。

### 追加

- **文字起こし言語の選択** — これまで日本語に固定していた文字起こし言語を、設定から選べるようにしました（Whisper が対応する言語。既定は日本語）。話者分離は言語に依存しません。
- **高精度な校正AIモデル（Gemma 4 12B）** — 設定タブの「AI校正バックエンド」から選べる、オプトインの上位モデルを追加しました。選択時に本体（約 7GB）を後からダウンロードします。既定は従来どおり Gemma 4 E4B（標準）で、選ばなければ動作は変わりません。

### 変更

- **校正モデルの選択方法を整理** — 専用の「校正AIモデル」セレクタを廃止し、「AI校正バックエンド」セレクタへ統合しました（標準 E4B / 高精度 12B / 外部 LLM アプリを1か所で選択）。
- **句読点の付与を言語に合わせて切り替え** — 日本語は全角（。、）、その他の言語は半角（. ,）を使うようにし、文末記号での分割処理も改善しました。
- **音声プレビューの再生速度** — ✕1.8 / ✕2.0 を選択肢から外し、上限を ✕1.6 にしました。
- 設定タブのコンテキスト長・並列処理の入力欄に、GPU VRAM に応じた目安のヒントを表示するようにしました。

### セキュリティ / プライバシー

- 外部 LLM アプリ（LM Studio / Ollama）との連携を **既定で無効**にしました。有効化は **インストール時の明示的なオプトイン**でのみ可能です（既定は「いいえ」）。アプリ内に再有効化のスイッチは設けておらず、有効／無効の変更には再インストールが必要です。
  - 無効時は LLM バックエンドの選択肢から LM Studio / Ollama を除外し、Rust 側でも OpenAI 互換 API 経路を拒否します（多層防御・フェイルクローズ）。
  - 内蔵 AI（Gemma 4 E4B / Lemonade）による校正はこの設定に関わらず常に利用できます。
- **オフライン動作の固定（フェイルクローズ）** — 文字起こし・話者分離の実行時に、モデル取得ライブラリをオフラインモードへ固定し、通常運用中に意図せず外部へ接続しないようにしました。モデルの取得は従来どおりセットアップ時に行います。
- **保存ファイルの暗号化パスワードの受け渡しを改善** — パスワードをコマンドライン引数ではなく標準入力経由で渡すようにし、同じ PC 上の他プロセスから見えないようにしました（暗号化を設定した場合のみ。未設定なら平文保存で従来どおり影響はありません）。
- **AI 校正の一時ファイルを確実に削除** — 校正で使う一時ファイルを、処理の成功・中止・失敗いずれの場合も確実に削除するようにしました（会話内容が一時ファイルとして残らないようにするため）。
- **フロントエンド依存の更新** — Angular を最新のパッチ版へ更新し、既知の脆弱性に対応しました。

### 修正・改善

- 校正対象のセグメントが無いときに校正エンジンを起動しないようにし、不要な VRAM 確保を防ぎました。
- プロセス終了時の VRAM 解放を CUDA / AMD 両構成で確実に行うよう改善しました。
- アンインストール時に、他のアプリと共有されうる汎用キャッシュ（`~/.cache/huggingface`）を操作しないようにしました。本アプリのモデル・データはアプリ専用フォルダ内で自己完結しており、アンインストールではこの領域のみを削除します。これに伴い、旧版向けの「Whisper モデルを削除しますか？」確認ダイアログを廃止しました。

### 開発・実験的構成向け（Windows CUDA / Editor 版には影響しません）

- AMD（ROCm）での高精度（12B）校正を **ROCm 優先 →（不可時）Vulkan フォールバック** に対応しました。MTP（投機的デコード）対応の新しい llama.cpp ビルドで高速化します（開発機の実測で約 35–37 tok/s）。
- 同梱 Lemonade を **10.8.0** に更新しました。プライバシー強化として LAN ブロードキャストを抑止（`no_broadcast`）し、10.8.0 で追加された Cloud offload は無効のままにしています（会話・音声データの外部送信は一切ありません）。
- MTP ドラフト併用時に FlashAttention を無効化し、一部の GPU / ビルドで起動時にクラッシュする問題を回避しました。

## 動作要件

- Windows 10 / 11 (x64)
- NVIDIA GPU + CUDA 12.x + cuDNN 9.x（CUDA 版。Editor 版は GPU 不要）
- VRAM 8GB 以上（CUDA 版。CPU のみでの動作は想定していません）
- ディスク空き容量: インストーラー約 1GB + モデルダウンロード分（高精度 12B モデルを使う場合は追加で約 7GB）

## インストールと初回セットアップ

1. インストーラーを実行する
2. アプリを起動し、セットアップタブから Python パッケージ・モデルをインストールする
   - **初回セットアップ時のみインターネット接続が必要です**（依存パッケージ・モデルの取得）
   - 話者分離モデルの取得には Hugging Face トークンが必要です（別 PC からのローカルコピーでも配置可能）
3. セットアップ完了後はオフラインで動作します

> **SmartScreen について**: 本インストーラーはコード署名されていないため、初回実行時に
> Windows SmartScreen の警告が表示されることがあります。「詳細情報」→「実行」で続行できます。
> 配布元（本 GitHub Release）からダウンロードしたファイルか、SHA-256 で確認してください。

## プライバシー

- 通常運用時はインターネットに接続しません
- 会話データ・音声データを外部 API へ送信しません
- LLM 校正を含むすべての推論はローカル（loopback）で完結します
- 外部 LLM アプリ（LM Studio / Ollama）との連携は既定で無効です（インストール時のオプトインでのみ有効化可能）

## 既知の問題

- AMD GPU（ROCm）版は experimental です。本リリースには含まれません
- インストーラーはコード署名されていないため SmartScreen 警告が表示されます（上記参照）
