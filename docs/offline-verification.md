# オフライン動作の確認手順（5分でできます）

本アプリ（Local Transcription for Therapy / LoTT）は「会話・音声データを PC の外へ送らない」ことを設計の中心にしています。
このページは、その主張を**利用者自身の目で確認する**ための手順です。技術的な知識は必要ありません。

> 考え方: 「送信していません（信じてください）」ではなく、**「インターネットに接続できない状態でも全機能が動きます（確かめてください）」**。
> クラウド型のサービス（ChatGPT 等）はネットを切ると動作しません。本アプリはネットを切っても動きます。これが「データが PC から出ていない」ことの最も分かりやすい証拠です。

## 前提

- 初回セットアップが完了していること（セットアップタブから Python パッケージと各モデルのダウンロードが済んでいる状態）。
  - セットアップ・モデルのダウンロード**時のみ**インターネット接続が必要です。このとき通信されるのは「プログラム部品と AI モデルの取得（受信）」であり、会話・音声データが送信されることはありません。
- 確認に使う音声ファイル（テスト用の録音で構いません）。

## 手順

1. **インターネット接続を切断します。** 次のいずれかで構いません。
   - ノート PC: 機内モードをオンにする
   - 有線接続: LAN ケーブルを抜く
   - Wi-Fi: 接続をオフにする
2. ブラウザで適当なページを開き、**本当にネットに繋がらないことを確認**します。
3. 本アプリを起動し、音声ファイルを選択して**文字起こし＋話者分離**を実行します。
4. 結果が表示されたら、**AI 校正**を実行します。
5. （音声入力パック導入済みの場合）編集画面で**マイク音声入力**や**区間聞き直し**を試します。
6. 結果を Word / Excel / JSON 形式で保存します。
7. すべて正常に完了すれば確認終了です。ネットワークを元に戻してください。

**期待される結果**: 手順 3〜6 のすべてが、オフラインのまま正常に動作します。

## 補足: 「わざと失敗させる」確認

本アプリは、必要なモデルが未取得の場合でも**勝手にインターネットへ取りに行かない**よう設計されています（フェイルクローズ設計）。
モデルを取得する前の状態でオフライン実行すると、自動ダウンロードで補われるのではなく「モデルが見つからない」というエラーで明示的に失敗します。「裏で黙って通信して動いてしまう」ことがない、という設計の確認になります。

## より厳密に確認したい方へ（任意・管理者向け）

一時的な切断では不安が残る場合、恒久的に通信を遮断した運用ができます。

- **Windows ファイアウォールで送信をブロックする**: 「セキュリティが強化された Windows Defender ファイアウォール」→「送信の規則」→「新しい規則」で、本アプリのインストールフォルダ配下の実行ファイル（本体 exe、`python.exe`、`llama-server.exe`、`ffmpeg.exe`）の送信接続をブロックします。この状態でも本アプリの全機能は動作します（モデルの追加ダウンロードだけができなくなります）。
- **通信監視ツールで観察する**: Wireshark、GlassWire、Sysinternals Process Monitor 等で、文字起こし・校正の実行中に本アプリ関連プロセスが外部アドレスへ接続しないことを観察できます。
- **ソースコードの確認**: 本アプリはオープンソースです。通信先を loopback（PC 内部、`localhost` / `127.0.0.1`）に限定する検証コードや、実行時のモデル取得ライブラリをオフラインモードに固定するコードを、リポジトリ上で直接確認できます。

## 注意事項

- 初回セットアップ、モデルの追加取得、アプリの更新時はインターネット接続が必要です。それ以外の通常運用でネット接続は不要です。
- OS・WebView ランタイム・GPU ドライバなど、本アプリ以外のシステム構成要素は独自に通信することがあります（Windows Update 等）。組織として完全なオフライン運用を求める場合は、OS やファイアウォール側での遮断を併用してください。
- データの保存先やバックアップの取り扱いなど、PC 側の管理については [プライバシー説明（非エンジニア向け）](privacy-guide.md) の「本アプリが守れないもの」も併せてお読みください。
