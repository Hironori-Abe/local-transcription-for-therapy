このプロジェクトの主要な変更点を記録します。

書式は [Keep a Changelog](https://keepachangelog.com/ja/1.1.0/) に、

各リリースの詳細・ダウンロード手順は `docs/release-notes-vX.Y.Z.md` を参照してください。

## [Unreleased]

### 修正

- AMD ROCmの文字起こしで、公式CTranslate2ホイールがネイティブ対応するgfx1102をgfx1100へ偽装していた `HSA_OVERRIDE_GFX_VERSION=11.0.0` の自動設定を削除。RX 7600M XTの10分データ比較で、既存JSONに対する文字編集距離が約15%減り、セグメント構成も参照結果に近づくことを確認した。MallocAsync回避用の `CT2_CUDA_ALLOCATOR=cub_caching` は維持する。
- 将来のCTranslate2 / ROCm更新後にgfx1102で不具合が出た場合、ネイティブarch互換性を最初に切り分ける手順をトラブルシューティングへ追記した。

## [0.9.5] - 2026-07-16

音声入力と区間聞き直しの速度・候補品質・低メモリ環境での案内を改善したリリース。

### 改善

- **音声入力モデルを再利用** — 初回処理後もモデルを保持し、2回目以降の音声入力・区間聞き直しを高速化。初回はモデルの読み込みに時間がかかることを状態欄に表示する。
- **候補の品質を改善** — 音声に忠実な候補、自然な表記、音として成立する別解を役割別に生成。区間聞き直しでは切り出し範囲に前後0.25秒の余白を加え、語頭・語尾の欠けを抑える。
- **候補操作を分かりやすく** — 挿入候補の動作説明を追加し、マイクボタンを大きくして押しやすくした。
- **Editor版のメモリ警告** — 16GB未満では編集画面の音声入力ボタンを初期非表示とし、設定タブから明示的に有効化できるようにした。16GB以上24GB未満では使用時の注意を表示し、24GB以上では警告しない。
- アンインストール時に、ダウンロード済みモデルを保持できるようにした。

### リソース管理

- 保持中の音声入力モデルは、校正・文字起こし・話者分離の開始時とアプリ終了時に解放する。
- Windowsでは音声入力用の推論プロセスをJob Objectに所属させ、アプリの強制終了時にも終了させる。

### セキュリティ・プライバシー

- **AIによるセキュリティ・プライバシー監査を実施** — コードベース全体を対象に、会話・音声データの外部送信経路の有無・オフライン設計の抜け道・一時ファイルへのデータ残留を監査（Anthropic Claude Fable 5、2026-07-16）。外部送信経路は発見されず、指摘された残余リスク4件（下記「監査指摘」印）を本バージョンですべて改修した。
- **公式版のローカルAIアプリ連携を無効化** — インストール時のLM Studio / Ollama連携ダイアログを廃止。公式ビルドでは常に無効とし、必要な利用者がCargo feature `local-llm-apps`付きでソースから専用インストーラーをビルドする方式に変更した。
- CSPを有効化し、画面からの通信先を同じPC内に制限した。本番ビルドからは開発専用の許可項目も除去した（監査指摘）。
- 編集画面の音声再生用ローカル配信にトークン認証を追加し、許可した音声ファイル以外にアクセスできないようにした。
- 会話内容を含む一時ファイルを、所有者のみアクセスできるアプリ専用領域に隔離し、処理後に自動削除するようにした。話者分離が作る一時音声ファイルも同じ領域に作成する（監査指摘）。
- 音声ファイルのパス等を、他プロセスから見えるコマンドライン引数ではなく環境変数で内部プロセスへ渡すようにした（監査指摘）。
- 接続先の「localhost」指定を実接続時に127.0.0.1へ固定し、hostsファイル改ざんへの耐性を持たせた。
- 話者分離モデル取得に使うHugging Faceトークンを、使用後にメモリ上から消去するようにした。
- 利用者・組織が自身で確認するための文書（オフライン動作の確認手順・プライバシー説明（非エンジニア向け）・倫理審査向け資料テンプレート）を追加した。

### その他

- アプリアイコンを更新した。

## [0.9.4] - 2026-07-10

編集画面に音声を使った2つの新機能（音声入力・区間聞き直し）を追加したリリース。どちらもオプトインの後付けダウンロード方式で、導入しなければ既定の動作は v0.9.3 と同じ。

### 新機能

- **音声入力（マイクから挿入候補を生成）** — 編集画面の各行のマイクボタンで最大15秒録音すると、ローカル AI（Gemma 4 E4B + 音声認識モジュール）が聞き取り、編集欄に挿入できる候補を最大3件提示。前後行の文脈は同じように聞こえる候補の表記選択にのみ利用。全エディション対応（CUDA / AMD = GPU 実行、Editor = CPU 実行）。
- **区間聞き直し（AI 再文字起こしで置き換え候補を生成）** — 各行の編集欄左側のボタンで、その行の時間範囲の音声を切り出して AI が聞き直し、「行の内容を置き換える候補」を最大3件提示。30秒超の行は先頭30秒のみ読み取り（実行時に通知）。Editor 版は音声切り出し用の LGPL ffmpeg（約95MB）を音声入力パックで後付けダウンロードし、導入時に LGPL 構成であることを自動検証する。
- **音声入力パック（後付けダウンロード）** — 設定タブから導入。CUDA / AMD 版は mmproj 約1GB（E4B 未導入時は追加で約4GB）、Editor 版は CPU バックエンド・ffmpeg 込みで合計約 5.1GB。
- **英語版 README（`README.en.md`）を追加** — 日本語版が引き続き正。

### 改善

- 音声入力・区間聞き直しのボタンはモデル未導入時もグレー表示され、ツールチップで設定タブへのダウンロードを案内する。
- 編集画面のボタン配置を整理（区間聞き直し＝編集欄左側、音声入力マイク＝編集欄右側、行下部は行コピー・句点分割の2ボタンを均等配置）。
- **対象言語を統一** — Whisper と Gemma 4 E4B 音声 ASR の共通12言語に整理。Editor 版にも言語選択を追加。

### 削除

- 編集欄右側の句読点追加ボタン（`+、` / `+。`）を廃止（音声入力ボタンの移動に伴う整理。AI 句読点付与・単純句読点付与の一括機能は従来どおり）。

### プライバシー

- マイク録音・切り出した音声区間はローカル推論エンジン（loopback 限定）のみで処理し、PC 外へ送信しない。音声入力パックの取得元は Hugging Face（モデル）と GitHub Releases（ffmpeg / llama.cpp バイナリ）のみで、必要なのはセットアップ時だけ。

### 内部変更

- 旧 Lemonade SDK への参照を除去し、サードパーティライセンス一覧を更新。
- Windows でのファイルダウンロードを curl 優先（PowerShell フォールバック）に変更。

## [0.9.3] - 2026-07-01

v0.9.2 以降のメンテナンス・改善リリース。既定の動作（日本語の文字起こし・Gemma 4 E4B による校正）は従来どおりで、設定を変えなければ v0.9.2 と同じように使えます。

### 変更・改善

- **文字起こしの進捗表示を滑らかに** — バックエンドからの離散的な進捗イベントの間を、推定所要時間に基づいて滑らかに補間して表示するようにしました（表示専用で、文字起こし処理の速度・内容には一切影響しません）。
- **全体校正カードの高さを調整** — 内容に応じて伸縮する（最大 85vh）ようにし、長い校正結果でも見やすくしました。

### セットアップ / モデル取得の改善

- **ダウンロード失敗時の案内を分かりやすく** — 話者分離モデルのダウンロードが失敗したとき、原因（ネットワーク不通・利用条件の同意反映待ち〈403〉・トークン不正〈401〉・アクセス権不足〈404〉・レート制限〈429〉・ディスク容量不足 など）に応じて、次に何をすればよいかを日本語で具体的に案内するようにしました。
- **Hugging Face トークンの形式チェック** — 送信前にトークンの形式（`hf_` で始まるか、空白・全角文字の混入、長さ）を検証し、打ち間違いを分かりやすく知らせます。
- **話者分離モデルの完全性判定を強化** — ダウンロードが途中で中断した際に残るマーカーを検出し、一部だけ揃った状態を正しく「未完了」と判定するようにしました。中断後はセットアップタブから不足分の補完ダウンロードを促します。

### 開発・実験的構成向け（Windows CUDA / Editor 版には影響しません）

- **同梱 Lemonade を撤去** — AMD（ROCm / Vulkan）での LLM 校正を、Lemonade デーモン / CLI 経由から llama.cpp `llama-server` の直起動へ統一しました。E4B・12B の両方で ROCm 優先 →（不可時）Vulkan フォールバックで動作します。CUDA 版は従来どおり同梱の CUDA llama-server を使うため影響ありません。
- **llama.cpp バックエンドのダウンロード対応** — AMD 用の `llama-server`（ROCm / Vulkan / CPU ビルド）を GitHub Releases から取得する仕組み（`download_llama_backend_cli.py`）を追加しました。取得元は公式リリースのバイナリのみで、会話・音声データを PC 外へ送信することは一切ありません。
- **Ubuntu 用 Docker ランチャースクリプトを追加** — AMD GPU / Tauri GUI の開発テスト用（`scripts/run-dev-docker-ubuntu.sh`）。

## [0.9.2] - 2026-06-23

v0.9.1 以降の機能追加とセキュリティ強化をまとめたリリース。既定の動作（日本語の文字起こし・Gemma 4 E4B による校正）は従来どおりです。

### 追加

- **文字起こし言語の選択** — これまで日本語に固定していた文字起こし言語を、設定から選べるようにしました（Whisper が対応する言語。既定は日本語）。話者分離は言語に依存しません。
- **高精度な校正AIモデル（Gemma 4 12B）** — 設定タブの「AI校正バックエンド」から選べる、オプトインの上位モデルを追加しました。選択時に本体（約 7GB）を後からダウンロードします。既定は従来どおり Gemma 4 E4B（標準）で、選ばなければ動作は変わりません。

### 変更

- **句読点の付与を言語に合わせて切り替え** — 日本語は全角（。、）、その他の言語は半角（. ,）を使うようにし、文末記号での分割処理も改善しました。
- **音声プレビューの再生速度** — ✕1.8 / ✕2.0 を選択肢から外し、上限を ✕1.6 にしました。
- 設定タブのコンテキスト長・並列処理の入力欄に、GPU VRAM に応じた目安のヒントを表示するようにしました。

### セキュリティ / プライバシー

- ローカルAIアプリ（LM Studio / Ollama）との連携を **既定で無効**にしました。有効化は **インストール時の明示的なオプトイン**でのみ可能です（既定は「いいえ」）。アプリ内に再有効化のスイッチは設けておらず、有効／無効の変更には再インストールが必要です。
  - 無効時は LLM バックエンドの選択肢から LM Studio / Ollama を除外し、Rust 側でも OpenAI 互換 API 経路を拒否します（多層防御・フェイルクローズ）。
  - 内蔵 AI（Gemma 4 E4B / Lemonade）による校正はこの設定に関わらず常に利用できます。
- **オフライン動作の固定（フェイルクローズ）** — 文字起こし・話者分離の実行時に、モデル取得ライブラリをオフラインモードへ固定し、通常運用中に意図せずインターネットへ接続しないようにしました。モデルの取得は従来どおりセットアップ時に行います。
- **保存ファイルの暗号化パスワードの受け渡しを改善** — パスワードをコマンドライン引数ではなく標準入力経由で渡すようにし、同じ PC 上の他プロセスから見えないようにしました（暗号化を設定した場合のみ。未設定なら平文保存で従来どおり影響はありません）。
- **AI 校正の一時ファイルを確実に削除** — 校正で使う一時ファイルを、処理の成功・中止・失敗いずれの場合も確実に削除するようにしました（会話内容が一時ファイルとして残らないようにするため）。
- **フロントエンド依存の更新** — Angular を最新のパッチ版へ更新し、既知の脆弱性に対応しました。

### 修正・改善

- 校正対象のセグメントが無いときに校正エンジンを起動しないようにし、不要な VRAM 確保を防ぎました。
- プロセス終了時の VRAM 解放を CUDA / AMD 両構成で確実に行うよう改善しました。
- アンインストール時に、他のアプリと共有されうる汎用キャッシュ（`~/.cache/huggingface`）を操作しないようにしました。本アプリのモデル・データはアプリ専用フォルダ（`%LOCALAPPDATA%\{識別子}`）内で自己完結しており、アンインストールではこの領域のみを削除します。これに伴い、旧版向けの「Whisper モデルを削除しますか？」確認ダイアログを廃止しました。

### ドキュメント

- README に「CPU のみでの動作は想定していない」「VRAM 8GB 以上が最低要件」である旨を明記しました。

### 開発・実験的構成向け（Windows CUDA / Editor 版には影響しません）

- AMD（ROCm）での高精度（12B）校正を **ROCm 優先 →（不可時）Vulkan フォールバック** に対応しました。MTP（投機的デコード）対応の新しい llama.cpp ビルドで高速化します（開発機の実測で約 35–37 tok/s）。
- 同梱 Lemonade を **10.8.0** に更新しました。プライバシー強化として LAN ブロードキャストを抑止（`no_broadcast`）し、10.8.0 で追加された Cloud offload は無効のままにしています（会話・音声データを PC 外へ送信することは一切ありません）。
- MTP ドラフト併用時に FlashAttention を無効化し、一部の GPU / ビルドで起動時にクラッシュする問題を回避しました。

## [0.9.1] - 2026-06-13

v0.9.0 からのメンテナンスリリース。アプリの基本機能（文字起こし・話者分離・校正）は v0.9.0 と同じです。

### 改善

- GPU を再確認するボタンを押した際に UI が一時的に固まる問題を改善（GPU・ランタイム判定をワーカースレッドへ移し、判定中も UI が応答し続けます）。

### 修正

- 文字起こし・話者分離・AI 校正（句読点付与／全体校正）の**二重起動ガード**を追加。実行中に再実行されても多重に GPU 処理が走らないようにしました。
- 起動時に表示されていた huggingface_hub の非推奨警告（`resume_download` / `local_dir_use_symlinks`）を解消。

### 開発・実験的構成向け（Windows CUDA / Editor 版には影響しません）

- 同梱 Lemonade を v10.7.0 に更新。コンテキスト長などの設定を環境変数から `config.json` へ移行（Lemonade 10.7.0 での設定用環境変数廃止に対応）。
- Linux 開発セットアップ（`scripts/setup-dev.sh`）の ROCm 向け ctranslate2 導入順を修正し、セットアップが途中で止まる問題を解消。
- AMD（ROCm）開発環境で同梱 lemond / lemonade を検出できるよう改善。
