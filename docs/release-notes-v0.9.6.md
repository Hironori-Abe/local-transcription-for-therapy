# Local Transcription for Therapy (LoTT) v0.9.6

臨床心理・カウンセリング会話のための、ローカル完結の日本語文字起こしデスクトップアプリです。
文字起こし・話者分離・文章校正を、会話データを PC の外へ送ることなく実行できます。

本バージョンでは、対応する GPU がない PC でも一連の動作を試せる CPU 版を追加しました。
あわせて、NVIDIA GPU 版の AI 校正を高速化し、AMD GPU 環境の互換性を改善しました。

## ダウンロード

| ファイル | 対象 | 備考 |
| --- | --- | --- |
| `LoTT-v0.9.6-windows-x64-cuda-setup.exe` | NVIDIA GPU (CUDA 12.x) | 主配布・安定版 |
| `LoTT-v0.9.6-windows-x64-cpu-setup.exe` | GPU 不要 | 動作確認・試用向け（常用非推奨） |
| `LoTT-v0.9.6-windows-x64-editor-setup.exe` | GPU 不要 | 校正・編集中心の軽量版（文字起こし・話者分離なし） |
| `SHA256SUMS.txt` | — | 各ファイルの SHA-256 チェックサム |

ダウンロード後の検証（任意）:

```sh
sha256sum -c SHA256SUMS.txt
```

## 変更点

### CPU 版（お試し用）を追加

- **GPU なしで文字起こし・話者分離に対応** — CPU だけで、音声の文字起こし、話者分離、単純な句読点付与を一連の流れとして実行できます。文字起こしの計算方式は `float32` が既定です。
- **編集機能も利用可能** — 文字起こし結果の確認・編集、音声再生、Word / Excel / JSON 保存を利用できます。
- **音声入力・区間聞き直しに対応** — 設定タブから音声入力パックを追加すると、Editor 版と同じく、マイク音声入力と行単位の区間聞き直しを CPU で利用できます。
- **AI 校正・全体校正は非搭載** — CPU 版の校正機能は、ローカルルールによる単純な句読点付与に限定しています。設定画面にも GPU 選択や AI 校正設定は表示しません。
- **CPU 専用の初回セットアップ** — PyTorch と文字起こし・話者分離ランタイムの CPU 版を導入し、校正用 LLM は基本セットアップの対象から外しました。
- **他のエディションから独立して管理** — CPU 版は専用のアプリ識別子と保存領域を使用するため、NVIDIA GPU 版や Editor 版の設定・ダウンロード済みモデルとは分けて管理されます。

CPU 版は処理時間が長いため、少量の音声で動作や文字起こし品質を確認する用途を想定しています。処理時間の目安は音声時間の約1.5〜2.5倍ですが、CPU 性能や音声内容によってはさらに長くなる場合があります。

開発機（Ryzen AI 9 HX 370、12コア / 24スレッド）では、11分43秒の音声について、CPU `float32` の文字起こしと話者分離の合計が約19分16秒（音声時間の1.64倍）でした。

### CPU 版の起動時チェック

- **最低要件を満たさない場合** — RAM 16GB未満、AVX2非対応、または8論理スレッド未満を検出すると、不足項目をダイアログに表示し、確認後にアプリを終了します。
- **最低要件を満たす場合** — CPU 版がお試し用であること、処理時間の目安、継続利用には GPU 版を推奨することを、起動のたびに案内します。
- メモリ容量を OS から取得できなかった場合は、誤判定による起動拒否を避け、CPU と論理スレッド数の判定を続行します。

### NVIDIA GPU 版の改善

- **AI 校正エンジンを更新** — 同梱の llama.cpp を b9571 から公式 CUDA 12.4 ビルド b10075 へ更新しました。
- **FlashAttention を再有効化** — Gemma 4 E4B / 12B と MTP の組み合わせでも FlashAttention を利用します。RTX 4060 Laptop 8GB・12B 長文テストでは、従来構成と比べて総処理時間を約37%短縮し、VRAM 使用量を約220MiB削減しました。
- **ローカル接続の制限を強化** — 同梱 llama-server が許可する Web origin を localhost に制限しました。Rust / Python からの loopback 接続は従来どおり利用できます。

Editor 版の CPU バックエンドと、AMD 版で後からダウンロードする llama.cpp は、従来どおり b9631 を使用します。

### AMD GPU 環境の修正

- CTranslate2 の公式 ROCm ホイールがネイティブ対応している gfx1102（RX 7600 系）を、gfx1100 として動作させる上書き設定を削除しました。
- RX 7600M XT の10分音声による比較では、既存結果に対する文字編集距離が約15%減り、セグメント構成も参照結果に近づきました。
- 将来の CTranslate2 / ROCm 更新後に問題が発生した場合に、GPU architecture の互換性を切り分ける手順をトラブルシューティングへ追加しました。

AMD GPU 版は引き続き experimental であり、本 Release の配布ファイルには含みません。

## 動作要件

### NVIDIA GPU 版

- Windows 10 / 11 (x64)
- NVIDIA GPU + CUDA Toolkit 12.x + cuDNN 9.x
- VRAM 8GB以上

### CPU 版

| 項目 | 最低要件 | 推奨要件 |
| --- | --- | --- |
| OS | Windows 10 / 11 64bit | Windows 11 64bit |
| CPU | AVX2対応、4コア / 8スレッド | 6コア / 12スレッド以上 |
| RAM | **16GB** | **24GB以上** |
| ディスク空き容量 | 約10GB | 約15GB以上 |
| GPU | 不要 | 不要 |

音声入力・区間聞き直しでは Gemma 4 E4B と音声認識モジュールも使用するため、RAM 24GB以上を推奨します。

## インストールと初回セットアップ

1. 利用するエディションのインストーラーを実行します。
2. アプリを起動し、セットアップタブから Python パッケージと必要なモデルをインストールします。
   - **初回セットアップ時のみインターネット接続が必要です**。
   - 話者分離モデルの取得には Hugging Face トークンが必要です。
   - 音声入力・区間聞き直しを使う場合は、設定タブから「音声入力パック」を追加します。
3. セットアップ完了後はオフラインで動作します。

> **SmartScreen について**: 本インストーラーはコード署名されていないため、初回実行時に
> Windows SmartScreen の警告が表示されることがあります。「詳細情報」→「実行」で続行できます。
> 配布元（本 GitHub Release）からダウンロードしたファイルか、SHA-256 で確認してください。

## プライバシー

- 通常運用時はインターネットに接続しません。
- 会話データ・音声データを PC 外の API へ送信しません。
- 文字起こし、話者分離、校正、音声入力、区間聞き直しを含む推論は、各エディションとも PC 内で完結します。
- インターネット接続を使用するのは、初回セットアップや追加モデル・バックエンドの取得時だけです。

## 既知の問題

- CPU 版の文字起こしと話者分離には、音声時間の約1.5〜2.5倍、またはそれ以上の処理時間がかかることがあります。
- CPU 版で音声入力・区間聞き直しを利用する場合、RAM 16GB環境では他のアプリを終了してから実行することを推奨します。
- AMD GPU（ROCm）版は experimental です。本 Release には含まれません。
- インストーラーはコード署名されていないため、Windows SmartScreen の警告が表示されます。
